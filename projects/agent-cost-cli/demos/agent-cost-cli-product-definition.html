01第一性原理：为什么要做这个工具

Agent 成本不再是黑盒。每一轮推理、每一次 LLM 调用，都是真金白银。

传统 API 调用成本追踪：查看账单 → 发现超支 → 手动排查哪个模型贵
Agent 成本追踪：cat agent_log.jsonl | agent-cost --frameworks all → 跨框架对比 → 架构决策有据可依

Agent 场景下成本问题更尖锐：

维度传统 API 调用Agent 推理成本
调用模式单次请求-响应多轮链式调用 + 工具调用 + 思考 tokens
定价单位简单 per-tokeninput + output + thinking + tool_call + cache 多重定价
框架选择固定供应商OpenAI vs Anthropic vs DeepSeek vs Google vs Mistral 可切换
成本隐匿账单可见agent loops 中 token 消耗难以预测

推导链

  1. Agent = Loop(Multi-turn Reasoning + Tool Calls + Context Accumulation)
  2. 每次 loop 产生 input tokens + output tokens + thinking tokens + tool call tokens
  3. 框架间定价差异巨大（Anthropic 靠 cache 降价 90%，DeepSeek 便宜但 thinking 爆量）
  4. 没有工具能在 CLI 管道级别做跨框架成本对比 — 企业选框架时只能拍脑袋
核心认知：「用 DeepSeek 替换 GPT-4o 省 80%」这种说法是错的。真实场景下 thinking tokens 可能吃掉所有节省。你需要的是：在同一个 log 上运行多个框架的定价计算，对比真实成本差异，才能做 informed decision。

时间窗口

  • 各框架定价持续变化 — Anthropic 推出 Prompt Caching, DeepSeek 频繁调价
  • Agent 框架碎片化 — LangChain, Vercel AI SDK, OpenAI SDK 各有日志格式
  • 多模型选型成常态 — 开发者不再绑定单一框架

共识：Agent 成本可视化是下一个必备 developer tool。没有它，你在盲飞。

03功能一览

核心能力：同一个 agent_log.jsonl，算 5 个框架的账，跨框架直接对比，秒级出结果。

核心功能（Phase 1 — 已上线）

功能描述命令
单框架计算解析 agent_log.jsonl 并计算指定框架成本--framework openai
多框架对比同一份 log，同时用 5 个框架计算并排对比--frameworks all
汇总模式只打印摘要，不展开 token 明细--summary
框架过滤指定对比哪些框架（逗号分隔）--frameworks openai,deepseek

定价颗粒度：不只是简单的 per-token 计算。每个框架的 input、output、thinking/base、cached、tool_call tokens 各有独立定价。Anthropic 的 Prompt Caching、DeepSeek 的 token compression、OpenAI 的 reasoning tokens — 全部有覆盖。

04技术架构

核心原语：Pricing Engine
token_count × price_per_token = cost
但在 Agent 场景下：input←∑turn_input, output←∑turn_output, thinking←∑turn_thinking, cache←∑turn_cache

代码结构

agent_cost/
├── cli.py                # Click CLI (3+ commands)
├── engine.py             # Cost calculation orchestrator
├── parser.py             # agent_log.jsonl parser
├── pricing.py            # Pricing table + per-framework calculation
├── compare.py            # Cross-framework comparison engine
├── formatter.py          # Terminal table / JSON output
└── pricing/
    ├── openai.py           # OpenAI pricing (5 models)
    ├── anthropic.py        # Anthropic pricing (3 models)
    ├── deepseek.py         # DeepSeek pricing (2 models)
    ├── google.py           # Google pricing (3 models)
    └── mistral.py          # Mistral pricing (3 models)
数据流：
agent_log.jsonl → Parser → Token Counts → Pricing Engine → Formatter → Terminal/JSON
跨框架 = 同一个 token_counts 经过 5 个不同的 pricing engine 独立计算。

Phase 1: Core CLI (✅ 已上线)

parser → agent_log.jsonl → structured token data
pricing_engine → framework × model pricing lookup
comparator → side-by-side multi-framework cost table
formatter → terminal table with severity coloring

Phase 2: Benchmark (🟡 WIP)

关键差异：

现存工具帮你看到花了多少钱。
Agent Cost CLI 帮你知道换一个框架能省多少钱。

LangSmith 说「你这个月花了 $12,000」。
Agent Cost CLI 说「如果用 DeepSeek，同样 log 只要 $890；用 Google 只要 $440。这里有 14 个模型的对比表。」

前者是记账。后者是决策支持。

差异化核心

🔀 Cross-Framework Compare

同一个 agent_log.jsonl，从 5 个框架 14+ 模型的视角同时算账。找最便宜的、最合适的。

🔧 CLI Pipeline First

cat log | agent-cost --frameworks all --summary --output json | jq — 纯 Unix 哲学。不对你的 stack 做任何假设。

📊 完整定价模型

不只是 input/output。thinking tokens、cache hits、tool call tokens — 每个框架的隐藏成本全部建模。

📈 Datadog Path

Phase 3 直接推 DogStatsD metrics → Datadog 面板。不依赖第三方 observability stack。
