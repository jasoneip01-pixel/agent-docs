⚡ Quick Start

Install

$ pip install agent-cost-cli

Single Framework Cost

$ cat agent_log.jsonl | agent-cost --framework openai

Cross-Framework Comparison

$ cat agent_log.jsonl | agent-cost --frameworks all

Summary Mode (compact)

$ cat agent_log.jsonl | agent-cost --frameworks all --summary

Compare Specific Frameworks

$ cat agent_log.jsonl | agent-cost --frameworks openai,deepseek

JSON Output (pipeline friendly)

$ cat agent_log.jsonl | agent-cost --framework openai --output json

❓ FAQ

What format does the input JSONL need to be?
Each line is a JSON object with agent call metadata: {"model": "gpt-4o", "input_tokens": 142580, "output_tokens": 38290, "thinking_tokens": 12450, "cached_input_tokens": 80120, "tool_call_tokens": 8300}. That's it. Any standard agent logging framework can produce this.
Is this just a calculator or does it actually call APIs?
It's a calculator. No API calls needed. It reads your agent log, looks up the per-token pricing for each framework, and computes the cost locally. Safe for offline use and CI/CD pipelines.
How is this different from LangSmith or Helicone cost tracking?
LangSmith/Helicone are observability tools — they show you what you already spent. Agent Cost CLI is a decision support tool — it shows you what you would have spent on alternative frameworks. Same log, 5 different cost views. It's for making framework switching decisions, not just accounting.
Is it open source?
Yes. MIT license. The pricing tables are community-maintained and updated when frameworks change their rates.
Can I add custom pricing?
Not yet — custom pricing is on the roadmap for v0.2.0. For now, all 14+ model prices are pre-loaded and versioned with each release.
How fast is it?
~10ms per framework for a typical agent log of 200-300 calls. The cross-framework comparison (all 5 frameworks) completes in under 50ms.
