01第一性原理：为什么 Agent 需要 MCP Registry

MCP 协议已有 1.64 亿次下载。Agent 的工具生态正在爆发。

传统软件生态：npm search express → npm install express → require('express')
Agent 工具生态：mcp-registry search gmail → mcp-registry install gmail-mcp → Agent 自动配置

MCP 生态的发现问题比 npm 更严重：

维度npm packageMCP Tool
发现方式npmjs.org · GitHub · 搜索引擎没有集中注册表
安装方式npm install手工 clone · 手写配置 · 无标准流程
版本管理semver · lockfile无版本概念
元数据package.json · README · 标签无标准化元数据格式
信任机制下载量 · 维护者 · 审计报告零信任信号

推导链

  1. MCP 协议正在标准化 Agent 工具接口 — 1.64 亿次下载不是巧合
  2. 工具越多，发现越困难 — GitHub 搜 "mcp-server" 已有数千仓库，没有分类和搜索
  3. 安装流程不统一 — 每个工具需要手动配置 JSON，没有标准 pipeline
  4. 需要 npm 级别的注册表体验 — search · install · version · publish
核心认知：MCP 协议解决了"Agent 如何调用工具"，但没解决"Agent 如何发现和安装工具"。MCP Registry 填补了这整条缺失的生态链。

时间窗口

  • MCP 下载量每月增长 30%+ — 从 2025 年初的 50M 到现在的 164M
  • Claude · Cursor · Windsurf · VS Code 全接入 — MCP 已是事实标准
  • npm registry 采集脚本已上线 — 每日自动发现新 MCP package

共识：Agent 工具注册表是 MCP 生态必须的基础设施。越早占据位置，越可能成为标准。

05技术架构

核心原语：MCP Registry 不是另一个 SaaS API，它是一个静态文件 + CLI
registry/index.json 是单一事实源（Single Source of Truth）
scripts/collect-mcp.py 每日更新这个文件
mcp-registry CLI 读取这个文件响应所有查询

代码结构

mcp-registry/
├── mcp_registry/
│   ├── cli.py              # Click CLI (5 commands)
│   ├── search.py           # Search engine (fuzzy + exact)
│   ├── installer.py         # Tool install + MCP config injection
│   └── publisher.py         # Tool publish workflow
├── registry/
│   └── index.json           # SSoT — 30 tools metadata
├── scripts/
│   └── collect-mcp.py      # Cron collector (GitHub + npm)
├── tests/
│   └── test_cli.py          # CLI test suite
└── pyproject.toml         # Python packaging

数据流

collect-mcp.py → GitHub API + npm registry 抓取
index.json → 每日更新 · git commit · git push
mcp-registry CLI → 读 index.json · 搜索 · 安装 · 发布
MCP Config → install 写到 ~/.config/mcp/servers.json

采集管道

GitHub Search → topic:mcp-server, stars:>100
npm Search → keyword:mcp-server
Dedup → 名称去重 + URL 归一化
Auto PR → 新增工具自动提 PR

部署拓扑

PyPI → pip install mcp-registry
GitHub Pages → 静态文档站 + 搜索 UI
GitHub Actions → 每日采集 cron
OAuth → GitHub OAuth for publish

08竞争定位

不做另一个 SaaS 聚合器，做 Agent 的 npm 注册表。

能力pulsemcpsmithery.aimcp.soawesome-mcpMCP Registry
CLI Search————✅

本地方案———✅✅ 唯一 CLI
每日采集——✅—✅ 开源脚本
离线可用————✅ 唯一
MCP Config 注入—手动——✅ 自动
开源———✅✅ MIT
差异化核心：CLI-first 的本地注册表体验。

所有竞品是 Web 聚合器（在浏览器里搜，复制 JSON 粘贴到配置文件）。
MCP Registry 是 npm 模式：terminal 里搜 → 一行命令安装 → 自动注入 MCP 配置。

不需要打开浏览器，不需要手写 JSON，不需要知道 MCP 配置格式。

差异化核心

⌨️ CLI-First

全部操作在 terminal 完成。不需浏览器，不需手写 JSON。

📦 自动安装

install 命令自动 clone + 配依赖 + 写入 MCP servers.json。零配置。

🔄 每日采集

scripts/collect-mcp.py 开源可复用。GitHub + npm 双源，你也能跑。

🔌 离线可搜

registry/index.json 是静态文件，clone 后离线全量搜索。与网络无关。
