为什么选择 MCP Registry

⌨️

CLI-First 体验

全部操作在 terminal 完成。搜索、安装、发布，无需打开浏览器。对标 npm search 的体验标准。

📦

一键安装 · 零配置

mcp-registry install postgres-mcp → 自动 clone + pip 依赖 + 注入 MCP 配置。到手即用。

🔄

每日自动采集

每日 02:00 UTC 从 GitHub + npm 双源发现新 MCP 工具，自动更新注册表。你永远有最新的工具。

🔌

离线可搜

registry/index.json 是静态文件。clone 仓库后离线全量搜索，零网络依赖。核心是文件，不是 API。

📋

元数据丰富

每个工具包含：版本、描述、类别、Stars、Tools 列表、依赖、许可证。结构化 JSON，可机器消费。

🤝

拥抱 MCP 生态

寄生 MCP 协议（164M 下载），兼容所有 MCP 客户端。Claude · Cursor · Windsurf · VS Code 全接入。

常见 问题

MCP Registry 和 pulsemcp / smithery.ai / mcp.so 有什么区别？
所有竞品是 Web 聚合器（浏览器搜 → 复制 JSON → 粘贴到配置文件）。MCP Registry 是 CLI-first：terminal 里搜 → 一行命令安装 → 自动注入 MCP 配置。不需要打开浏览器，不需要手写 JSON，像 npm 一样工作。
数据源从哪里来？更新频率？
GitHub (topic:mcp-server, stars>100) + npm (keyword:mcp-server) 双源。每日 02:00 UTC 自动采集，GitHub Actions cron 调度，自动 dedup + 自动 PR。
离线能使用吗？
可以。clone 仓库后 mcp-registry search 使用本地的 registry/index.json，全量搜索零网络请求。线上模式会拉取最新 index.json。
如何发布自己的 MCP 工具？
mcp-registry publish ./my-tool。CLI 会工具目录中的 mcp_registry.json，验证后提交 PR 到注册表仓库。审核合并后全球可用。
支持哪些 MCP 客户端？
任何实现了 MCP 协议的客户端。Claude Desktop、Cursor、Windsurf、VS Code (GitHub Copilot)、自定义 MCP Agent 等。install 命令自动写入 ~/.config/mcp/servers.json。
路线图下一步是什么？
Phase 3: GitHub Pages 文档站 (进行中)。Phase 4: install 命令完整实现 (自动 pip + 配置注入)。Phase 5: OAuth 集成 (GitHub 登录发布)。Phase 6: 社区生态 (评分、信任、私有注册表)。
