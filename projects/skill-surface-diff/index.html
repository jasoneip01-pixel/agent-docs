⚡ Quick Start

Install

$ pip install skill-surface-diff

Diff Two Versions

$ skill-surface-diff diff old/ new/

Scan + Save Baseline

$ skill-surface-diff scan ./skill/ --save-baseline --tag v1.8.1

Diff Against Baseline

$ skill-surface-diff diff --baseline v1.8.1 ./skill-v1.8.2/

SARIF / JSON Output

$ skill-surface-diff diff old/ new/ --format sarif -o diff.sarif
$ skill-surface-diff diff old/ new/ --format json

List Baselines

$ skill-surface-diff baselines

🛡️ 4 分析维度

Manifest

📋 Manifest Diff

检测 SKILL.md 中新增/删除的字段、system_prompt_append 等非标准字段、能力声明 (network/files/tools) 的变更。v1.8.1 到 v1.8.2 多了什么，少了什么。

Script

⚙️ Script Diff

检测新增/删除/修改的脚本文件。识别新脚本中的 data exfiltration、shell 命令变化、新加的可执行文件。防止恶意代码在版本更新中潜入。

Permission

🔑 Permission Diff

检测网络权限 (network:true)、文件系统范围、工具权限声明的增减。network:true 突然出现？文件读取范围从 .env 扩展到 /etc/secrets/*？立刻标记。

Tool Schema

📐 Tool Schema Diff

检测工具声明的新增/删除/修改：参数变化 (optional→required)、安全 guardrails 丢失、命名空间变化。新增 payment.refund 而 guardrails 不全？阻断。

Policy Gate

🎯 规则优先阻断

规则引擎优先于风险评分。设定 permission.expansion: block 则直接阻断。风险评分 (加权公式) 只用于优先级排序，不改变阻断决策。

CI/CD

🔄 CI/CD Gate

GitHub Action 触发→基线 diff→SARIF 上传→PR 自动注释。阻止不合规的版本合并。Phase 3 规划中：Slack 通知、跨 repo 仪表盘。

⚖️ 风险评分公式

加权评分：Rule-based 阻断先行，评分仅用于优先级排序

维度权重说明
Permission Expansion (PSD)40新增网络/文件/工具权限 — 攻击面最大增量
Manifest Injection (PID)15新增非标准字段 / 指令注入尝试
Script Changes (SCD)8新增/修改脚本中的危险操作
Tool Schema Changes (TSD)5Schema 结构变更 / guardrails 丢失
Network Expansion5网络范围扩大 / 无理由新增 network:true

⚠️ 风险评分是辅助信号，不是决策依据。 规则引擎的 block/warn 决定是否阻断合并。评分的唯一价值是：有多个警告时，按分数确定修复优先级。

❓ FAQ

How is this different from Skill Scanner?
Skill Scanner scans a single skill for malicious content. Surface Diff compares two versions and says "this version adds X, Y, Z capabilities." They're complementary: scanner for absolute risk, diff for relative risk. Best used together.
How is this different from Snyk's mcp-scan?
Snyk checks if a skill is toxic right now. Surface Diff checks if a skill version expanded the agent's authority beyond what the previous version allowed. Snyk doesn't do version-to-version diff. This is the only tool that does.
Why rule-based blocking over risk score?
Risk scores are informative, but a scoring algorithm shouldn't decide whether to block a merge. Rules like permission.expansion: block are explicit, auditable, and reviewable. The risk score helps prioritize when multiple warnings exist. This reflects the "policy-as-code" principle.
What does a baseline look like?
A baseline.json stored in .skill-surface/baseline/<tag>/. Contains full capability surface extraction: manifest fields, script hashes, permission sections, tool schemas. Cached, versioned, reusable across CI runs.
Is it open source?
Yes. MIT license. 2 rounds of adversarial security audits completed on the scanning engine. Security tools must be transparent about their detection logic.
Can it integrate with my CI pipeline?
Phase 3 (planned Q3 2026) adds GitHub Action, SARIF upload to Code Scanning, automatic PR comments, and Slack notifications. For now, the CLI returns exit codes compatible with any CI.
