01第一性原理：为什么需要跨框架 Agent 可观测性

你的 Agent 不只用一家 API。那为什么监控工具只盯一家？

传统 APM 生态：Datadog → 多语言/多框架 APM · 全栈可见
Agent 可观测性生态：AgentWatch → 单框架监控 · 无跨模型对比

AI Agent 的成本面比传统 API 监控复杂得多：

维度传统 APM (Datadog)现有 Agent 监控Agent Datadog
覆盖范围多语言/多框架单一框架5 框架 + 跨模型对比
成本维度基础设施 + API单一模型定价框架对比 + Token/Budget 分析
延迟分析全链路追踪单一调用路径多框架并行对比
决策支持基础设施优化单一供应商"用哪个 API 最划算"

推导链

  1. Agent 系统 = 多个模型 API + 多个框架 SDK + Prompt 编排层 + 工具调用层
  2. 团队选型时需要回答：OpenAI vs Claude vs DeepSeek — 哪个性价比最高？
  3. 现有工具只监控单一家——AgentWatch 只看 OpenAI，LangSmith 只看 LangChain
  4. 跨框架对比需要统一的定价模型 + 统一的性能基准 + 统一的仪表盘
  5. 不回答"哪个最便宜"的可观测性，只是漂亮图表
核心认知：Agent 的边际成本 = 调用次数 × Token量 × 模型单价。每天百万次 call、五个候选模型的场景下，10% 的成本差异 = 几千美金/月。跨框架对比不是锦上添花，是 ROI 的决策杠杆。

时间窗口

  • 多模型路由成为标配 — OpenAI → 降价 → Anthropic → 追加 → DeepSeek → 开源 → Google → 捆绑
  • Agent 框架碎片化 — LangChain / Vercel AI SDK / OpenAI SDK / Claude API / 自研 — 没人统一看
  • 成本成为产品决策 — Agent 不再 demo-only，生产环境的成本压力是真实需求

共识：跨模型、跨框架的 Agent 可观测性是空白市场。AgentWatch 证明了单框架有需求，agent-datadog 证明了跨框架有差异化。

04技术架构

核心原语：Cross-Framework Cost Surface
cost_surface = [benchmark(f) for f in frameworks]
best_route = argmin(f.cost_per_100k_tokens * tokens + f.avg_latency_ms * weight + f.error_rate_pct * penalty)

05API 参考

GET /api/cost/compare?model={model_name}

查询指定模型在所有框架中的成本对比。返回各框架中与该模型同级别的所有可选方案。

参数类型必须说明
modelstring是模型名称，如 gpt-4o, claude-3.5-sonnet
frameworkstring否限定对比框架，如 openai
min_scorenumber否最低综合评分 (0-10)

GET /api/cost/benchmark

全量基准数据。返回所有框架 × 所有模型的定价、延迟、错误率、综合评分。

参数类型必须说明
limitnumber否返回条目数 (默认 14)
sort_bystring否排序字段: cost, latency, score
frameworkstring否按框架过滤

GET /events

SSE 实时事件流。仪表盘通过 EventSource 订阅，实时展示成本/Latency/错误率变化。

// JavaScript
const source = new EventSource('/events');
source.addEventListener('cost_update', (e) => {
  const data = JSON.parse(e.data);
  updateDashboard(data);
});

AgentWatch 已验证需求，agent-datadog 填补空白。

AgentWatch 证明了"Agent 需要可观测性"这个市场。但 AgentWatch 只做单框架。当你的 Agent 同时用 3 个模型、2 个框架、5 个 API 时——谁帮你比谁更划算？

AgentWatch answers: "Is my agent healthy?"
agent-datadog answers: "Which agent setup costs me the least?"

差异化核心

🔄 Cross-Framework

5 框架 14 模型统一对比。输出 "best value" 推荐，而不是让用户自己查定价页。

💰 Cost as a Decision Tool

不只是好看图表。回答: "换成 deepseek-chat 在同等质量下能省多少钱？"

📊 Weighted Scoring

综合评分 = 成本 40% + 延迟 30% + 可靠性 30%。可调权重的决策框架。

🔌 Datadog Ecosystem

与现有 Datadog 工作流集成。Agent 指标进入统一 APM 视图。
